Part A: Identify + Fix (Unique Mistakes)

Issue Type Mistake Example Correction / Solution
Missing Alt Text <img src="logo_final.jpg"> Every image needs an alt attribute so screen readers can describe it.

<img src="logo_final.jpg" alt="Company Logo">
Vague Link Text <a href="info.html">click here</a> Links should describe the destination clearly instead of using generic text like "click here".

<a href="info.html">View Course Information</a>
Table for Layout <table><td>Nav</td><td>Main</td></table> Tables should only be used for tabular data or comparisons, not for page layout.

<main><nav>...</nav></main>
Incorrect Tag Use <section>Home About Contact</section> Specific semantic tags like <nav> should be used for navigation areas.

<nav>Home About Contact</nav>
Missing Main Tag <body><div>Main content here</div></body> The primary content of the document must be wrapped in a <main> tag.

<main>Main content here</main>
Decoration Alt <img src="border.png" alt="border"> Decorative images should use an empty alt string (alt="") so screen readers ignore them.

<img src="border.png" alt="">
Header for Size <h1>Copyright Info</h1> Don't use heading tags just to change font size; they are for document hierarchy.

<p style="font-size: 20px;">Copyright Info</p>
Semantic Header <div class="header">My Site</div> Use the <header> element for site or section headers to improve structure.

<header>My Site</header>
Semantic Footer <div class="footer">2026</div> Use the <footer> element specifically for footer content.

<footer>2026</footer>
Article Misuse <article>Navigation Menu</article> The <article> tag is for self-contained, independent content, not navigation.

<nav>Navigation Menu</nav>

Part C: Short Answer

Explain the difference between a semantic element (e.g., nav) and a generic element (e.g., div).

A semantic element, such as <nav>, explicitly describes its purpose to the browser and developer, whereas a generic element like <div> has no inherent meaning. This improves site accessibility by providing landmarks for screen readers and makes the code easier to maintain. While <div> is useful for general styling, semantic elements like <header> and <main> ensure the document structure is clear and meaningful.

Why does KLM not model high-level planning and interpretation?

The Keystroke-Level Model (KLM) is designed to predict the performance of expert users executing routine, well-practiced tasks. It assumes error-free performance, meaning the user already knows the steps and does not need to pause for decision-making. Since it focuses on low-level physical actions like typing or clicking, it excludes the cognitive time required for learning or complex planning.
