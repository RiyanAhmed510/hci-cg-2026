Lecture 1: Introduction to HCI Concepts

Defining HCI

Human–Computer Interaction (HCI) combines human cognition, design principles, and software engineering to create systems that are intuitive, efficient, and easy to use.

Behavior Patterns and Habits

Humans often act based on ingrained habits or learned behaviors. For instance, when sketching familiar objects, people usually rely on memory rather than observing in detail. While habits can evolve, underlying tendencies often persist.

Types of Interfaces

  • Graphical User Interface (GUI) – interact through visual elements
  • Command Line Interface (CLI) – interact via text commands
  • Voice Interface – interaction using speech
  • Gesture Interface – interaction using physical movements

Memory Systems

  • Short-Term Memory (STM): holds limited information for a short period
  • Long-Term Memory (LTM): retains information over long durations
  • Sensory Memory: briefly retains input from senses such as sight or sound

Cognitive Processing

Cognitive processing allows humans to recognize patterns, organize information, and respond intelligently to tasks. Understanding these processes helps designers create systems that align with natural human thought.

Fitts’ Law

Fitts’ Law predicts the time required to reach a target based on its size and distance. It guides interface design and the creation of devices like the computer mouse.

Chunking Technique

Chunking organizes information into smaller, meaningful units to make it easier to remember. For example, writing a phone number as 123-456-7890 instead of a long string of digits.

Case Study: Medical Device Interface

Small interface flaws can lead to major consequences. For example, confusing key layouts in medical devices can result in incorrect inputs. Iterative prototyping, such as paper mock-ups, helps designers identify and fix issues early.

This demonstrates the importance of testing and refining designs in a controlled, iterative manner.

Lecture 3: Norman’s Action Model and Gulfs

Gulf of Execution and Gulf of Evaluation

The Gulf of Execution is the challenge users face when turning intentions into system-supported actions.

The Gulf of Evaluation is the difficulty users experience when interpreting system feedback to verify goal achievement.

Norman’s Seven-Stage Action Cycle

Users proceed through stages: forming goals, planning actions, executing them, perceiving results, interpreting outcomes, and evaluating success. Interface design should support each stage to reduce errors and improve usability.

Example: Group Expense App

  • Cognitive Load: Users track individual expenses and totals, adding mental effort.
  • Social Pressure: Discussing payments is uncomfortable, so accurate app calculations are essential.
  • Feedback Clarity: Clear display of totals helps users understand current status.
  • External Cognition: Visual representation reduces mental strain.

Goal: reduce stress and ensure accurate expense splitting.

Execution Gulf: Difficulty entering detailed expense data.
Evaluation Gulf: Difficulty confirming totals match actual payments.
